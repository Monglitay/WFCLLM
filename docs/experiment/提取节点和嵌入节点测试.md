# 任务目标
我们需要编写一个诊断实验脚本，用于排查 WFCLLM 项目中“代码水印生成（Embedding）与提取（Extraction）效果不一致”的 Bug。

# 背景上下文
目前的系统表现是：在 `generator.py` 中生成含水印代码时，能够成功嵌入水印；但将生成的代码送入 `detector.py` 进行检测时，水印命中率极低。
我怀疑在**生成端（流式 AST 事件）**和**提取端（静态 Tree-sitter 解析）**之间存在严重的**非对称性 (Asymmetry)**。主要怀疑以下三个方面：
1. **文本边界差异 (Text Normalization)**：生成端截获的 `block_text` 和提取端 `tree_sitter` 截取的 `block.source` 在空格、缩进、换行符上可能存在细微差异，导致 LSH 签名 (u) 突变。
2. **父节点类型推断差异 (Parent Node Type)**：生成端的 `event.parent_node_type` 和提取端 `scorer.py` 中 `_resolve_parent_type` 解析出的类型可能不一致，导致生成 `valid_set` 的密钥空间偏移。
3. **Compound/Simple Block 的过滤不对称**：生成时 Cascade 和 Fallback 机制可能会在 Compound Block 上成功嵌入水印，但提取端目前只检测 Simple Block。
